# /update - 更新專案進度

當使用者說「更新專案」或使用 /update 指令時，請執行以下動作：

## 自動執行步驟

1. **讀取目前狀態**
   - 讀取 `PROJECT_STATUS.md`
   - 讀取 `PROJECT_LOG.md`

2. **詢問更新內容**（如果使用者沒有提供）
   - 今天完成了什麼？
   - 遇到什麼問題？
   - 下一步要做什麼？

3. **更新 PROJECT_STATUS.md**
   - 更新「最後更新」時間
   - 更新整體進度百分比
   - 移動已完成項目到「已完成」區
   - 更新「進行中項目」
   - 更新「已知問題」（如有）
   - 更新「下次工作重點」

4. **更新 PROJECT_LOG.md**
   - 在最新日期下新增一條記錄
   - 包含：時間、事件、變更內容、原因（如適用）

5. **回報更新結果**
   - 顯示更新摘要
   - 顯示目前整體進度

## 範例輸出

```
✅ 專案已更新！

📊 整體進度：15% → 25%

今日完成：
- ✅ Next.js 專案初始化
- ✅ Tailwind CSS 設定

進行中：
- 🔄 Prisma 資料庫設定

下次重點：
- 完成資料庫設定
- 開始對話介面 UI

📝 日誌已更新：PROJECT_LOG.md
```
